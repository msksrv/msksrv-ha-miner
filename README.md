# MSKSRV ASIC Miner — интеграция Home Assistant для ASIC-майнеров

**Язык:** **Русский** · [English](README.en.md)

**MSKSRV ASIC Miner** — это пользовательская **[интеграция для Home Assistant](https://www.home-assistant.io/)** для **локального мониторинга и управления ASIC-майнерами Bitcoin** (Antminer, WhatsMiner, Avalon, Innosilicon, Goldshell, IceRiver, BitAxe и другие, поддерживаемые [**pyasic**](https://github.com/UpstreamData/pyasic)). Установка через **[HACS](https://www.hacs.xyz/)** или вручную; **одна запись конфигурации на IP майнера**, при необходимости учётные данные **RPC / веб / SSH**, инструменты для **stratum-пула** и опциональное устройство **«ферма»**, которое суммирует хешрейт, мощность и позволяет **массово применять настройки пула** ко многим майнерам. Для фермы доступен **опциональный учёт стоимости электроэнергии**: **плоский** тариф (до трёх валют) или **две/три зоны** по **местному времени** Home Assistant.

## Возможности {#features}

### Настройка и обнаружение

| Возможность | Описание |
|-------------|----------|
| **Обнаружение по DHCP** | Home Assistant сопоставляет **имена хостов в нижнем регистре** с шаблонами (`whatsminer*`, `antminer*` и т.д.) и/или известные **префиксы MAC (OUI)**; затем интеграция опрашивает API майнера с ограниченным числом повторов (см. [Обнаружение](#discovery)). |
| **Сканирование подсети** | Сканирование IPv4-подсети (CIDR) с индикатором прогресса; выбор найденного хоста. Размер подсети ограничен из соображений безопасности. |
| **Ручной ввод IP** | Добавление майнера по адресу. |
| **Ферма** | Устройство второго уровня: выбор существующих **майнеров** → **суммарный хешрейт (TH/s)**, **суммарная мощность (кВт)**, **число майнеров / онлайн**, сводка по **алгоритму**, **аварийная остановка** (вызывает `switch.turn_off` для **реле питания**, заданного через **⚙️ Настроить** на плитке **каждого** участника-майнера). Через **⚙️ Настроить** на плитке **фермы** можно применить **один и тот же основной или резервный stratum** ко всем участникам, если они сообщают один алгоритм. Опционально — **тарифы на электроэнергию** (плоско или **2/3 зоны** по местному времени) и датчики **стоимости** по суммарной мощности. |
| **Учётные данные** | По необходимости: RPC, веб-интерфейс и SSH, если майнер эти API предоставляет. |
| **Реле питания и пул (опции)** | В **⚙️ Настроить** на плитке майнера: привязка **`switch`** для розетки (**Отключить питание** / **Включить питание**), по желанию **заменить основной stratum** или **добавить резервный пул** (хост, порт, SSL, воркер), когда майнер в сети. |

### Мониторинг (датчики)

Интервал опроса — **10 секунд** (`local_polling`). Типичные данные:

- **Производительность** — хешрейт, идеальный хешрейт, имя активного пресета (если доступно), эффективность (Дж/Тх)
- **Питание** — потребление, лимит мощности (зеркало только для чтения на датчиках, где применимо)
- **Температура** — средняя; **по платам** — температура платы, чипов, хешрейт платы
- **Чипы** — число чипов по платам, ожидаемое число, работающие, эффективный процент
- **Вентиляторы** — обороты (RPM) по каждому вентилятору
- **Пул** — хост/порт основного пула, **воркер пула** (пользователь stratum из активного или первого слота), принятые/отклонённые шары, доля отказов
- **Устройство** — модель, прошивка, время работы (форматированное), число плат, IP, MAC
- **Ферма (по желанию)** — если в **⚙️ Настроить** на записи фермы заданы тарифы: датчики **стоимости электроэнергии** (текущий час, сутки, месяц, накопительно и **стоимость при текущей мощности**) в выбранной валюте; в режиме зон цена берётся по **местному времени** HA.

При кратковременной недоступности майнера интеграция **сохраняет сущности**, на первом сбое подставляет **ослабленные / нулевые** данные, при повторных сбоях помечает устройство как недоступное.

### Управление (платформы)

| Платформа | Назначение | Когда появляется |
|-----------|------------|------------------|
| **`switch`** | **Активность** — пауза / возобновление майнинга (`stop_mining` / `resume_mining`). | Майнер сообщает `supports_shutdown`. |
| **`number`** | **Лимит мощности** (ватты, с шагом). | Майнер сообщает `supports_autotuning`. |
| **`select`** | **Режим питания** — низкий / нормальный / высокий. | Режимы питания поддерживаются **и** автотюнинг **не** используется (как в pyasic). |
| **`select`** | **Приоритет пулов** — перестановка слотов так, чтобы выбранный стал основным. | В конфигурации майнера не менее **двух** пулов в основной группе. |
| **`button`** | **Перезагрузка** — перезагрузка устройства через pyasic. | Всегда (на каждый майнер). |
| **`button`** | **Отключить питание** — `switch.turn_off` для реле, выбранного через **⚙️ Настроить** на плитке майнера. | Кнопка всегда в списке; **доступна** только при корректно настроенном и существующем в HA `switch`. |
| **`select`** | **Ферма: пресет для применения** — какой сохранённый слот stratum используют кнопки фермы. | Только устройство **ферма**; есть хотя бы один пресет. |
| **`button`** | **Ферма: применить пресет как основной / резервный пул** — та же логика, что массовый stratum через **⚙️ Настроить** на плитке фермы; используется выбранный пресет. | Устройство **ферма**; нужны пресеты. |

### Автоматизация

- **Службы** — перезагрузка, перезапуск бэкенда майнинга, режим работы, пул, пул фермы (см. [Службы](#services)).
- **Действия устройства** — перезагрузка, перезапуск бэкенда, режим работы из UI автоматизаций (см. [Действия устройства](#device-actions)).
- **Примеры YAML** — [Примеры автоматизаций (YAML)](#automation-examples-yaml).

## Установка {#installation}

### HACS (рекомендуется)

1. Откройте **HACS → Интеграции**.
2. **Пользовательские репозитории** — только если репозиторий ещё не добавлен: укажите  
   `https://github.com/msksrv/msksrv-ha-miner`, категория **Integration**.
3. Либо нажмите кнопку **«Открыть репозиторий в HACS»** вверху этого README.
4. Установите **MSKSRV ASIC Miner** и **перезапустите Home Assistant**.

### Вручную

Скопируйте каталог `custom_components/miner` в каталог  
`config/custom_components/` вашего Home Assistant, затем перезапустите систему.

## Настройка {#configuration}

### Значки из интерфейса Home Assistant

Ниже те же условные обозначения, что помогают найти пункты в UI (иконки в боковой панели и на карточках):

| Значок | Куда нажать |
|:------:|-------------|
| ⚙️ | **Настройки** (глобально) — **шестерёнка** в боковой панели (часто внизу слева). Отличается от **⚙️ Настроить** на строке интеграции ниже. |
| 🔌 | **Устройства и службы** — раздел внутри настроек. |
| 🧩 | Вкладка **Интеграции** внутри «Устройства и службы» — **не** вкладка «Устройства». |
| ➕ | **Добавить интеграцию** на экране интеграций. |
| **⚙️ Настроить** | На **плитке записи** *MSKSRV ASIC Miner* в **🧩 Интеграциях** (одна строка на IP или **ферму**): пункт **Настроить** или иконка **шестерёнки** на строке (иногда через меню **⋯**). Откроются **опции интеграции** — это **не** карточка устройства во вкладке «Устройства». |
| 👤 | **Профиль** — меню пользователя (аватар / инициалы), оттуда настройки **боковой панели**. |
| 🛠️ | **Инструменты разработчика** — вкладки **Устройства** (для `device_id`), **Действия** (вызов служб). |

1. **⚙️ Настройки → 🔌 Устройства и службы → 🧩 Интеграции → ➕ Добавить интеграцию → MSKSRV ASIC Miner.**
2. Выберите **Сканирование**, **Вручную**, **Ферма** (агрегат) или завершите поток после **обнаружения по DHCP**.
3. При запросе введите **RPC / веб / SSH**, если они нужны.
4. Задайте **имя устройства** (удобно для зон и панелей).

### Опции интеграции (питание и stratum)

**⚙️ Настройки → 🔌 Устройства и службы → 🧩 Интеграции** (это **не** вкладка «Устройства»). Найдите **MSKSRV ASIC Miner**, выберите **запись этого майнера** (одна строка-плитка на IP), затем **⚙️ Настроить**.

- **Реле питания** — выбор только среди **`switch`**. **Отключить питание** / **Включить питание** остаются недоступными, пока не сохранён корректный выключатель (или сущность удалена). Очистите поле и сохраните, чтобы отвязать.
- **Stratum-пул** — **ничего не делать**, **задать основной пул** или **добавить резервный пул**, затем **хост**, **порт**, при необходимости **SSL** и **воркер**. Майнер должен быть **доступен** при отправке формы; в основной группе до **3** пулов (добавление не сработает, если слоты заполнены).

Поля пула применяются **только при отправке** этой формы; долгосрочно в опциях не хранятся (кроме идентификатора сущности реле питания).

### Боковая панель и панели

Используйте **встроенные** средства Home Assistant: **👤 Профиль** → настройки **боковой панели**; либо представление **Lovelace** с карточками майнеров и пункт **Показывать в боковой панели**. Эта интеграция **не** регистрирует отдельную панель (чтобы не дублировать меню).

### Устройство «ферма» {#farm-device}

Добавьте **Ферма** из того же меню интеграции, укажите **имя** и выберите несколько **майнеров** (только записи одиночных майнеров, не другие фермы).  
**Аварийная остановка** **доступна**, только если у хотя бы одного участника через **⚙️ Настроить** на **его** плитке майнера задано **реле питания** и этот `switch` есть в HA. Отправляется **`switch.turn_off`** на **все** такие реле (параллельно, без блокировки).

Чтобы **добавить или убрать майнеры** у существующей фермы, откройте **⚙️ Настроить** на плитке **записи фермы** (в **🧩 Интеграциях**), обновите список **Устройства-майнеры** и сохраните. Новых майнеров сначала добавьте отдельными записями интеграции, чтобы они появились в списке.

В **⚙️ Настроить** на плитке **фермы** также можно привязать **датчики температуры помещения**: один или несколько **`sensor`** (например ZigBee). Каждый отображается на устройстве фермы; **имя** совпадает с дружественным именем исходной сущности (единица — как у источника). Сохранение перезагружает запись.

**Stratum (все участники)** — до **пяти** слотов-пресетов на **запись интеграции фермы** (у каждой фермы **свой** список в опциях). В слоте: хост, порт, SSL, воркер, пароль. **Очистите хост** (и порт) слота и сохраните, чтобы **удалить** пресет. Массовое применение из **⚙️ Настроить** на плитке **фермы** — выбор **«какой слот (1–5)»** и **заменить основной** / **добавить резервный**; поведение как у **`miner.set_pool`**. **Каждый** привязанный майнер должен принять изменение, иначе форма выдаст ошибку. Массовое применение **блокируется**, если участники сообщают **более одного алгоритма** (по последнему успешному опросу).

- **Воркер** — для слота те же правила: общая строка или шаблоны **`{ip}`** / **`{ip_last}`** (только для фермы). У многих **Bitcoin**-пулов принят формат `субсчёт.имя_воркера` (аналог subaccount.workername); для других монет смотрите документацию пула.
- **Панель** — на **карточке устройства** **фермы** (вкладка **Устройства**): **Пресет для применения** (`select`) и кнопки **Применить пресет как основной пул** / **… как резервный** — **без** **⚙️ Настроить** в интеграциях. При ошибке (разные алгоритмы, майнер офлайн) в журнал пишется сообщение.
- **Секреты** — пресеты хранятся в **конфигурации HA** вместе с записью фермы; учитывайте это в модели угроз для **резервных копий**.

**Электроэнергия (стоимость)** — в **⚙️ Настроить** на плитке **фермы**, блок учёта: режим **плоский** (до трёх пар «валюта + цена за кВт·ч») или **две / три зоны** по **местному времени** Home Assistant (одна валюта; у каждой зоны — начало, конец и цена за кВт·ч). Конец суток удобно задавать как **24:00**. Сенсоры стоимости создаются только при корректно заполненной конфигурации выбранного режима. После **смены режима тарифа или зон** выполните **перезагрузку записи интеграции** фермы (**🧩 Интеграции** → строка фермы → меню **⋯** → перезагрузить), чтобы набор сущностей совпал с настройками.

Датчики фермы: **суммарный хешрейт / мощность**, **число майнеров / онлайн**, **сводка по алгоритмам** (из pyasic по каждому майнеру; при различии, например `SHA256d (3), Scrypt (1)`; если алгоритм не сообщён, подпись **SHA256d** условная), **эффективные чипы %** (сумма работающих к ожидаемым по **онлайн**-участникам), привязанные **датчики температуры окружающей среды** и при настроенных тарифах — **стоимость электроэнергии** (см. выше).

## Сущности {#entities}

### Датчики (примеры)

Имена вида `{название устройства} …`; номера плат и вентиляторов зависят от железа.

| Область | Примеры |
|---------|---------|
| Майнер | Хешрейт, идеальный хешрейт, активный пресет, температура, лимит мощности, потребление, эффективность |
| Диагностика | IP, MAC, модель ASIC, прошивка, время работы, число плат |
| Пул | Хост, порт, **воркер пула** (пользователь stratum), принятые/отклонённые шары, доля отказов |
| По плате | Температура платы, чипов, хешрейт платы, чипы / ожидаемые / рабочие / % |
| По вентилятору | Скорость (об/мин) |

### Другие платформы

- **`switch.*_active`** — майнинг вкл/выкл, где поддерживается.
- **`number.*_power_limit`** — потолок автотюнинга в ваттах, где поддерживается.
- **`select.*_power_mode`** — низкий / нормальный / высокий, где поддерживается.
- **`select.*_pool_priority`** — выбор основного пула среди настроенных слотов (≥ 2 пулов).
- **`button.*_reboot`** — программная перезагрузка.
- **`button.*_power_off`** / **`button.*_power_on`** — `switch.turn_off` / `switch.turn_on` для привязанного реле.

## Примеры автоматизаций (YAML) {#automation-examples-yaml}

Подставьте свои `device_id` из **🛠️ Инструменты разработчика → Устройства** (или выберите устройство в редакторе автоматизаций — подставится корректный id).

### Перезагрузка одного майнера при включении переключателя-помощника

```yaml
automation:
  - alias: "Перезагрузка майнера по переключателю"
    triggers:
      - trigger: state
        entity_id: input_boolean.reboot_miner_now
        to: "on"
    actions:
      - action: miner.reboot
        data:
          device_id:
            - abc123yourDeviceIdHere
      - action: input_boolean.turn_off
        target:
          entity_id: input_boolean.reboot_miner_now
```

### Задать основной stratum-пул (ручной режим)

```yaml
automation:
  - alias: "Новый основной пул для майнера"
    triggers:
      - trigger: time
        at: "04:00:00"
    actions:
      - action: miner.set_pool
        data:
          device_id:
            - abc123yourDeviceIdHere
          mode: manual
          host: "pool.example.com"
          port: 3333
          use_ssl: false
          username: "account.worker1"
          password: "optional_pool_password"
```

### Ферма: один пул на всех привязанных майнерах

В строке воркера можно использовать **`{ip}`** или **`{ip_last}`** для каждого майнера (например `mypool.{ip_last}`).

```yaml
automation:
  - alias: "Смена пула фермы"
    triggers:
      - trigger: state
        entity_id: input_select.farm_pool_choice
    actions:
      - action: miner.set_farm_pool
        data:
          device_id:
            - farmDeviceIdHere
          mode: manual
          host: "stratum.pool.com"
          port: 443
          use_ssl: true
          username: "user.{ip_last}"
          password: ""
```

## Службы (services) {#services}

Все службы принимают **`device_id`** (идентификатор устройства Home Assistant). Службы для майнеров поддерживают **несколько** майнеров за один вызов; **`miner.set_farm_pool`** — для устройств **фермы** (одно или несколько).

| Служба | Описание |
|--------|----------|
| **`miner.reboot`** | Перезагрузка майнера. |
| **`miner.restart_backend`** | Перезапуск процесса майнинга на устройстве. |
| **`miner.set_work_mode`** | `mode`: `low` \| `normal` \| `high`. |
| **`miner.set_pool`** | **Существующий слот** — `pool_index` (с нуля) становится основным (перестановка). **Ручной** — задать основной слот: `host` / `port` / при необходимости `use_ssl`, `username`, `password`. **Добавить** — резервный слот (те же поля; максимум 3 пула). |
| **`miner.set_farm_pool`** | **`device_id`**: устройство(а) **фермы**. **`mode`**: `manual` (заменить основной у всех) или `append` (резервный слот). Те же **`host` / `port` / `use_ssl` / `username` / `password`**, что у `set_pool`. **Без действия с записью в журнал**, если алгоритмы **различаются** или майнер недоступен / отклоняет изменение. |

Вызывайте из **🛠️ Инструменты разработчика → Действия** или YAML-автоматизаций. Поведение пулов зависит от **прошивки / модели** и поддержки pyasic для `get_config` / `send_config`.

## Действия устройства {#device-actions}

Для автоматизаций, привязанных к **устройству** (не к сырым entity_id), интеграция предоставляет:

- Перезагрузка  
- Перезапуск бэкенда майнинга  
- Режим работы (с выбором режима)

`set_pool` и **`set_farm_pool`** в **действиях устройства** **не** доступны; вызывайте **`miner.set_pool`** или **`miner.set_farm_pool`** из автоматизаций.

## Обнаружение {#discovery}

DHCP использует **оба** пути (как в v1.2.x): шаблоны **имени хоста** только в **нижнем регистре** — Home Assistant приводит имя к нижнему регистру до `fnmatch`, например `WhatsMiner` → `whatsminer*` (шаблон `WhatsMiner*` **не** совпадёт). Плюс префиксы **MAC OUI** (**Bitmain** `E0A509`, класс **WhatsMiner** `C41025` / `C80831`) для пустых или общих имён, и **`registered_devices`**, если MAC уже в реестре устройств.

При совпадении интеграция обращается к API майнера **до 3 раз** (таймаут 6 с, паузы **2 с / 5 с** между попытками). Если ответа нет, открывается поток обнаружения с **предзаполненным** IP для ручного завершения. **Уже настроенные** майнеры пропускаются по **IP**, **MAC в реестре** и **`unique_id`** (только не фермы). При необходимости добавьте строки `hostname` / `macaddress` в `manifest.json` → `dhcp`.

## Требования {#requirements}

- **Home Assistant** ≥ **2025.1.0** (см. `hacs.json`).
- **Зависимость Python**: **pyasic** `0.78.x` (диапазон в [`manifest.json`](custom_components/miner/manifest.json); устанавливается Home Assistant автоматически).

Если в журнале **`Requirements for miner not found`** для pyasic: хост должен иметь доступ к **PyPI** (или прокси). На очень новых сборках HA (**Python 3.14**) колёса могут отставать; обновление Home Assistant или повтор через сутки часто помогает. В манифесте указано **`pyasic>=0.78.9,<0.79`**, чтобы pip мог выбрать сборку, которая ставится без ошибок.

Интеграция подгружает **pyasic только когда нужно** (настройка одиночного майнера, скан, службы и т.д.). Настройка **фермы** и меню потока конфигурации могут работать, даже если pyasic ещё не установился — сначала устраните установку pyasic, чтобы снова загружались **майнеры по IP**.

## Устранение неполадок {#troubleshooting}

| Симптом | Что проверить |
|---------|----------------|
| **Майнер недоступен** | Ping IP; пароли RPC/веб; файрвол; веб-интерфейс майнера с хоста HA. |
| **Нет значения у датчика воркера пула** | Прошивка/API может не отдавать пользователя stratum в статистике пула; при наличии данных используется основной/активный пул. |
| **Применение пула фермы срывается на одном участнике** | В журнале строки `Farm stratum:` — офлайн, разные алгоритмы или дубликаты записей устройств на ферме (исправьте список устройств через **⚙️ Настроить** на плитке **фермы**). |
| **DHCP-обнаружение не срабатывает** | Шаблоны имён в `manifest.json` — в **нижнем регистре**; HA приводит имена DHCP к нижнему регистру перед сопоставлением. |
| **Лишний пункт боковой панели «MSKSRV…» после обновления** | В старых бета-версиях регистрировалась своя панель; отключите в **👤 Профиль → боковая панель** или перезапустите HA после обновления до **1.4.x**. |
| **Нет датчиков стоимости у фермы** | Укажите тарифы в **⚙️ Настроить** на записи фермы. В режиме **две/три зоны** нужны валюта и столько зон с **положительной** ценой, сколько требует режим. После изменений — **перезагрузка записи** интеграции фермы. |
| **Ошибки PyPI / установки pyasic** | Сеть, корпоративный прокси, версия Python; см. [Требования](#requirements). |

**Задачи и предложения:** [GitHub Issues](https://github.com/msksrv/msksrv-ha-miner/issues).

## Ограничения {#limitations}

- **Не у каждого майнера есть все функции** — переключатели, режимы питания, автотюнинг, настройка пулов и датчики зависят от драйвера в pyasic и прошивки.
- **Смена пулов** — при нескольких пулах используйте **Приоритет пулов**; произвольные URL — через **`miner.set_pool`** в ручном режиме.
- **Отключение / включение питания** — только через **привязанный HA `switch`**; не заменяют API отключения на стороне майнера.

Семейства устройств, поддерживаемые pyasic:

https://pyasic.readthedocs.io/en/latest/miners/supported_types/

## Релизы {#releases}

**Текущая стабильная линейка: 1.6.x** — учёт **стоимости электроэнергии** для фермы: режим **плоский** (до трёх валют) и **две/три зоны** по **местному времени**; расчёт по суммарной мощности с учётом границ зон; обновлены строки интерфейса настроек (английский и русский). Ранее в **1.4.x–1.5.x**: stratum фермы, датчик **воркера пула**, правки потока настроек и DHCP.

### Автоматический релиз (GitHub Actions)

Отправьте тег версии:

- **Стабильный:** `v1.6.0` (или следующий semver) → полноценный **Release**, вложение `miner.zip`, в workflow версия в `manifest.json` приводится к тегу (без ведущего `v`).
- **Бета / RC:** `v1.7.0b1`, `v1.7.0rc1` и т.д. → **Pre-release** (тот же ZIP).

Workflow: **Create release from tag** (черновик релиза) и **Release** (после публикации: правка версии в артефакте, архив `custom_components/miner`, загрузка **`miner.zip`**).

### Вручную

Можно создать релиз в интерфейсе GitHub: выберите тег, при необходимости **Pre-release** для бет, затем **Publish** — ZIP-сборка выполняется так же.
